Development Roadmap

Timeline and Current Status of Sparkle Protocol

Last Updated: December 2025

1. Current Status

Status: Core Protocol Implemented

TypeScript reference implementation complete and verified. Taproot atomic swaps working. Ready for testnet deployment.

Sparkle Protocol has completed its core implementation phase. The Sparkle Swap mechanism is now a working TypeScript library with full Taproot script support, claim and refund transaction builders, and comprehensive test coverage.

1.1 Completed Work

  • Specification v1 drafted and published
  • Sparkle Swap trustless atomic swap mechanism designed
  • Metadata standards for Lightning-aware Ordinals defined
  • Documentation website published
  • TypeScript reference implementation complete
  • Taproot hashlock + timelock scripts verified
  • Claim and refund transaction builders working
  • All cryptographic tests passing

1.2 Recently Completed

  • Lightning invoice integration (BOLT11 decoding)
  • NIP-07 wallet integration (Alby, nos2x)
  • Bitcoin wallet integration (Unisat, Xverse)
  • Secure frontend (sparkle-swap.js) - no private key input
  • CLI tool for testnet swap execution

1.3 Next Steps

  • First manual testnet swap execution
  • Production coordinator implementation
  • Marketplace integrations

4. Contributing

Sparkle is an open protocol seeking contributors, feedback, and early adopters. There are several ways to participate:

4.1 For Developers

Review the specification, suggest improvements, and build integrations. The complete technical specification is available at spec.html and the source code is hosted on GitHub.

4.2 For Wallet and Marketplace Operators

Consider supporting Sparkle checkout in your platform. The Coordinator API documentation provides integration details.

4.3 For Community Members

Provide feedback on the specification, test demonstrations when available, and help spread awareness of the protocol.
